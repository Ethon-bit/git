# 上海地铁路径规划与运营管理系统

## 项目简介

基于 **Dijkstra 算法** 的上海地铁网络路径规划系统，支持最短时间路径查询、最少换乘路径查询、K条最优路径、站点运营状态管理等功能。

- **编程语言**: C++14
- **运行环境**: Windows 控制台
- **数据规模**: 11条地铁线路、328个站点、738条边

## 项目结构

```
metro_system/
├── data/
│   ├── Station.csv              # 站点数据（ID、名称、线路、状态）
│   ├── Edge.csv                 # 边数据（起止站、类型、方向、时间）
│   └── update_station_status.csv # 批量更新站点状态示例
├── main.cpp                     # 主程序源代码
├── build.bat                    # Windows 一键编译脚本
├── Makefile                     # Linux/Mac 编译
└── README.md
```

## 快速开始

### Windows

双击 `build.bat` 编译，然后运行 `metro.exe`

或手动编译：
```cmd
g++ -std=c++14 -O2 -o metro.exe main.cpp
metro.exe
```

### Linux/Mac

```bash
make
./metro
```

## 功能说明

### 主菜单

| 选项 | 功能 |
|------|------|
| 1 | 线路站点信息/运营状态管理 |
| 2 | 所需时间最短路径规划 |
| 3 | 所需换乘次数最少路径规划 |
| 4 | 退出系统 |

### 运营状态管理

- 从CSV文件批量更新站点开启/关闭状态
- 手工更新站点状态（支持模糊搜索）
- 显示当前关闭站点清单
- 恢复所有站点初始状态
- 查询线路全部站点详情（含换乘信息）

### 路径规划

- **最短时间路径**: Dijkstra 算法，考虑换乘步行时间
- **K条最优路径**: Yen's 算法，返回前K条最短路径
- **最少换乘路径**: 改进 Dijkstra，以换乘次数为第一优先级

### 特殊支持

- 4号线环线（内圈/外圈标记）
- 5号线支线（东川路→闵行开发区）
- 11号线支线（嘉定新城→花桥）
- 站内换乘与虚拟换乘（如上海火车站1/3/4号线）

## 算法说明

### Dijkstra 最短路径
采用优先队列优化的 Dijkstra 算法，时间复杂度 O((V+E)log V)。

### 最少换乘路径
在 Dijkstra 基础上修改边权：换乘边权重×100000 + 时间权重，优先最小化换乘次数。

### K条最短路径
基于 Yen's 算法，通过偏离路径思想迭代生成候选路径。
